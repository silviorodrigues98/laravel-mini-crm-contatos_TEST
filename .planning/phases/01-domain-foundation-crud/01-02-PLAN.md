        4. Call `$contact->changePhone(new Phone($phone))`.
        5. Call `$this->repository->save($contact)`.
        6. Return the updated Contact.

    C) `src/Application/UseCases/DeleteContactUseCase.php`:
      - Namespace `Application\UseCases`. Constructor receives `ContactRepositoryInterface`.
      - Method `execute(int $id): void`

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-06 | Tampering | PUT /api/contacts/{id} | mitigate | `$fillable` on Eloquent model; UpdateContactRequest uses `sometimes` + validated(); UpdateContactUseCase only applies fields through entity methods (updateName, updateEmail, changePhone) |
| T-01-07 | Tampering | DELETE /api/contacts/{id} | accept | Soft delete only (deleted_at set, row preserved). No hard delete available. This is intended behavior per spec (soft-delete only) |
| T-01-08 | Spoofing | Update email validation | mitigate | UpdateContactRequest uses `email:rfc` format validation + `Rule::unique()->ignore($this->route('contact'))->whereNull('deleted_at')` |
| T-01-09 | Information Disclosure | GET /api/contacts/{id} | accept | No auth per spec. Contact data is intentionally public. Soft-deleted contacts return 404 (information about existence is intentionally hidden per REST conventions) |
| T-01-10 | Tampering | Update non-existent contact | mitigate | UpdateContactUseCase throws \DomainException if findById returns null; Laravel converts to 404 response |
</threat_model>
